LDAP Server Deployment & Splunk Integration Guide

This guide covers how to deploy a lightweight LDAP server on AWS EC2 and configure Splunk to use it for user authentication.

Step 1: Deploy LDAP Server on EC2

We have automated the entire installation process. You only need to run one command.

1. Connect to your EC2 Instance

SSH into your Amazon Linux EC2 instance:

ssh -i "your-key.pem" ec2-user@<your-ec2-ip-address>

2. Run the Auto-Install Command

Copy and paste this entire line into your terminal:

sudo bash -c 'yum update -y; yum install git -y; git clone https://github.com/splunklearners1/adminldap.git; cd adminldap; chmod +x autoldap.sh; ./autoldap.sh'

What this does:

  • Installs Git and Docker
  • Clones the LDAP project repository
  • Builds the Docker container
  • Starts the LDAP service on port 1389
  • Starts the Web UI on port 3000
Verify Installation:
Open your browser to: http://<your-ec2-ip>:3000
If you see the LDAP User Manager dashboard, the deployment was successful!

Step 2: Configure Splunk Integration

You have two ways to configure Splunk: manually via UI or automatically via config file.

Method A: The "Config File" Way (Fastest)

1. Copy the authentication.conf file provided in the installation folder.

2. Edit the file and replace host = 172.31.9.197 with your actual LDAP Server IP.

3. Place the file on your Splunk server at:

$SPLUNK_HOME/etc/system/local/authentication.conf

(Linux: /opt/splunk/etc/system/local/ | Windows: C:\Program Files\Splunk\etc\system\local\)

4. Restart Splunk:

$SPLUNK_HOME/bin/splunk restart

Method B: The "Splunk Web UI" Way (Manual)

1. Go to Settings > Authentication Methods.

2. Select LDAP and click Configure Splunk to use LDAP.

3. Add a new strategy with these settings:

Setting Value
Host YOUR_EC2_IP
Port 1389
Bind DN cn=ldapadmin,ou=users,o=splunk
Password password
User Base DN o=splunk
User Name uid
Real Name cn
Email mail
Group Base DN o=splunk
Static Member Attr member
Group Name Attr cn

4. Map Groups: Go to "Map groups" and link them:

  • SplunkAdmins → admin role
  • SalesUserGroup → user role

Step 3: Test Authentication

Now, verify that users can log in.

1. Test Credentials

Use these pre-configured users to test different access levels:

Username Password Group Expected Role
salesuser1 password123 SalesUserGroup user (Basic Access)
ldapadmin password SplunkAdmins admin (Full Access)

2. Verification Steps

  1. Open Splunk in Incognito/Private mode.
  2. Enter Username salesuser1 and Password password123.
  3. Click Login.
  4. Success: You should be logged in!
  5. Check your role: Click your name in top bar > verify it says "user".

Troubleshooting

  • Cannot connect to LDAP? Ensure Security Group for EC2 allows Inbound Traffic on port 1389 and 3000.
  • Login fails? Check splunkd.log for "LDAP" errors.
  • Wrong role? Check your Group Mappings in Splunk Settings.
